System Architecture & Data Flow

The system is built using a clean **Decoupled Model** separating the pathfinding core (`algorithms.ts`) from the React 19 rendering layer, ensuring smooth performance even during 50-iteration background benchmarks.

M
Algorithmic Pathfinding Core (`algorithms.ts`) Processes raw grids dynamically and outputs sequential arrays of visited nodes to maximize CPU speeds and cache hits.
V
Real-time Grid Rendering (`GridRenderer.tsx`) Renders 900 responsive nodes in real-time using CSS hardware acceleration, supporting freehand multi-target drawing via mouse actions.
C
Independent Adversarial Laboratory (`ScenarioLab.tsx`) Runs a parallel virtual sandbox environment to execute and visualize two search algorithms simultaneously in real-time.

Procedural Terrain Generation
Applies dynamic mathematical sine waves paired with random noise to procedurally generate organic, winding rivers and swamp lanes (`maze.ts` -> `generateSwampTerrain`).
maze.ts TypeScript 
const sineWave = Math.sin(col * 0.3) * 4 + 10;
const distanceToWave = Math.abs(row - sineWave);
 
if (distanceToWave < 2.2) {
  weight = 10; // Deep water (extreme resistance)
} else if (distanceToWave < 4.5) {
  weight = 5;  // Swamp mud (moderate drag)
}
Principle: By calculating the Euclidean distance from coordinates to the sine wave centerline, we generate fluid, natural water bodies procedurally.

Heuristic Estimates & Diagonal Movement

To enable standard 8-directional movement, we integrate four heuristic distances along with geometric collision controls to prevent clipping through diagonal obstacle corners (**Corner-Crossing Control**).

1. Manhattan Distance Designed for 4-way orthogonal movement (up, down, left, right).
2. Octile Distance Perfect for 8-directional movement with diagonal scale of sqrt(2).
3. Chebyshev Distance Used when diagonal and orthogonal movements have uniform cost.
algorithms.ts (Obstacle Corner-Crossing Logic) TypeScript 
// Check adjacent diagonal nodes in algorithms.ts
if (allowDiagonal) {
  // Case: Up - Left
  if (up && left) {
    const target = grid[row - 1][col - 1];
    const blocked = dontCrossCorners
      ? (upNode?.isWall || leftNode?.isWall)
      : (upNode?.isWall && leftNode?.isWall);
    if (!blocked) neighbors.push(target);
  }
}
💡 Technical Insight: The `dontCrossCorners` option blocks diagonal paths when either of the adjacent orthogonal cells is a wall, fully resolving the common "wall clipping" visual glitch in game AI programming.

Implementing the Parallel Adversarial Laboratory

We successfully engineered a parallel simulation engine inside `ScenarioLab.tsx`. Utilizing a synchronized `setInterval` heartbeat matching the visual speed, the sandbox handles step-by-step state changes on two fully independent grid clones.

Core advantages of our laboratory architecture:
  • Deep-cloned matrices maintain zero side effects on the primary playground data.
  • Automatically aggregates terrain cost step weights matching custom playback speeds (1x to 10x).
  • Visually differentiates expanding fronts (BFS Wavefront in blue, Dijkstra in dark purple) using hardware-accelerated CSS filters.
